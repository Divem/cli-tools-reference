PENCIL CLI
参考手册

Pencil.dev 推出的 AI 驱动设计文件操作命令行工具。通过自然语言 Prompt 直接生成或修改 .pen 设计文件，支持交互式 Shell、批量任务、多模型选择和图片导出，适合将 AI 设计能力集成到开发工作流中。

版本：0.2.4
API：https://api.pencil.dev
格式：.pen
$ npm install -g @pencil.dev/cli
📦

单次命令模式

通过 --prompt 参数一次性生成或修改设计文件。

完整参数

参数短选项说明
--in <path>-i输入 .pen 文件（省略则为空白画布）
--out <path>-o输出 .pen 文件路径（必填）
--prompt <text>-pAI Agent Prompt（必填）
--model <id>-m指定模型（默认 Claude Opus）
--custom-c使用自定义 Claude 模型配置
--list-models列出可用模型并退出
--tasks <path>-tJSON 批量任务文件
--workspace <path>-wAgent 运行的工作区目录
--export <path>-e导出最终结果图片
--export-scale <n>导出缩放倍数（默认 1）
--export-type <type>导出格式：png/jpeg/webp/pdf（默认 png）
--verbose-mcp在响应中包含完整 MCP 工具错误详情

示例

bash 
# 从零创建设计
pencil -o landing.pen -p "Create a landing page with hero section"

# 基于现有文件修改
pencil -i existing.pen -o modified.pen -p "Add a footer with social links"

# 指定模型
pencil -o form.pen -p "Create a login form" -m claude-sonnet-4-6

# 生成并导出图片
pencil -o design.pen -p "Create a pricing page" -e ./exports/pricing.png --export-scale 2

交互式 Shell interactive

交互式 Shell 是 Pencil CLI 的核心模式，通过 MCP 工具直接操作 .pen 文件，适合复杂设计任务。

启动方式

bash 
# 连接桌面应用
pencil interactive --app desktop -i design.pen

# 无头模式：新建文件
pencil interactive -o output.pen

# 无头模式：编辑现有文件
pencil interactive -i input.pen -o output.pen

Shell 内置命令

tool_name({ ... })
调用 MCP 工具（JS 对象参数）
save()
保存文档到磁盘
exit()
退出交互式 Shell
💡 文件路径自动注入，无需手动传递 filePath 参数。

batch_design 操作详解

核心设计操作工具，支持在一次调用中执行多个插入/复制/更新/替换/移动/删除/图片生成操作。

⚠ 每次调用最多 25 个操作，超出应拆分为多次调用。任一操作失败则全部回滚。

Insert (I) — 插入节点

javascript 
nodeId = I(parent, nodeData)

// 插入一个容器 Frame
hero = I(document, {type:"frame", name:"Hero", width:1440, height:900, fill:"#0A0A0A"})

// 在 hero 中插入文本
title = I(hero, {type:"text", content:"Ship faster.", fontSize:72, fill:"#FFFFFF"})

// 插入组件实例
card = I(body, {type:"ref", ref:"CardComp"})

Copy (C) — 复制节点

javascript 
copiedId = C(path, parent, copyNodeData)

// 复制并偏移定位
dashboardV2 = C("Xk9f2", document, {name:"Dashboard V2", positionDirection:"right", positionPadding:100})

// 复制组件并修改后代属性
label1 = C("NKYzH", container, {descendants:{"ZopUS/jEYMs":{content:"First Name"}}})

Update (U) — 更新节点

javascript 
U(path, updateData)

// 更新普通节点
U("MmNEt", {layout:"horizontal", gap:16})

// 更新组件实例的子节点
U("submit-button/label", {content:"Submit"})

Replace (R) — 替换节点

javascript 
replacedId = R(path, nodeData)

newTitle = R("cardInstance/headerSlot", {type:"text", content:"Custom Title"})

Move (M) — 移动节点

javascript 
M(nodeId, parent, index?)

// 移到文档根级
M("frameId")

// 移到指定父节点末尾
M("frameId", "parentId")

// 移到指定位置
M("frameId", "parentId", 0)

Delete (D) — 删除节点

javascript 
D(nodeId)

// 删除指定节点
D("oldFrame")

Generate Image (G) — 生成图片

javascript 
G(nodeId, type, prompt)

// 创建 frame 后应用 AI 图片
heroImg = I("parentId", {type:"frame", name:"Hero Image", width:400, height:300})
G(heroImg, "ai", "modern office workspace bright")

// 使用素材图
G("bg-frame", "stock", "mountain landscape")

batch_get 查询详解

参数

参数类型说明
nodeIdsstring[]按节点 ID 读取
patternsobject[]搜索模式列表
parentIdstring限定搜索范围
readDepthnumber读取深度（默认 1）
searchDepthnumber搜索深度（默认无限制）
resolveInstancesboolean展开 ref 节点为完整结构
resolveVariablesboolean显示变量计算值

搜索示例

javascript 
// 列出文档顶级节点
batch_get()

// 查找所有可复用组件
batch_get({ patterns: [{ reusable: true }], readDepth: 2, searchDepth: 3 })

// 按名称搜索
batch_get({ patterns: [{ name: "Button" }] })

// 读取特定节点
batch_get({ nodeIds: ["frameId1", "frameId2"], readDepth: 3 })

其他 MCP 工具

get_editor_state

javascript 
get_editor_state({ include_schema: true })

首次操作必须调用并传入 include_schema: true 加载 .pen 文件 schema。

get_guidelines

javascript 
// 列出所有指南和样式
get_guidelines()

// 加载特定指南
get_guidelines({ category: "guide", name: "Landing Page" })

// 加载特定样式
get_guidelines({ category: "style", name: "Aerial Gravitas" })

export_nodes

javascript 
export_nodes({
  nodeIds: ["hero", "sidebar"],
  outputDir: "./exports",
  format: "png",
  scale: 2
})

find_empty_space_on_canvas

javascript 
find_empty_space_on_canvas({ width: 1440, height: 900, padding: 100, direction: "right" })
find_empty_space_on_canvas({ width: 400, height: 300, padding: 50, direction: "bottom", nodeId: "frameId" })

snapshot_layout

javascript 
// 检查整体布局
snapshot_layout({ maxDepth: 2 })

// 只查看有问题的布局
snapshot_layout({ problemsOnly: true })

导出功能

常用命令示例

单次命令

bash 
# 从零创建落地页
pencil -o landing.pen -p "Create a modern SaaS landing page with hero, features, pricing, footer"

# 修改现有设计
pencil -i dashboard.pen -o dashboard-v2.pen -p "Change to dark mode, update all text to white"

# 使用低成本模型快速迭代
pencil -o wireframe.pen -p "Create a low-fidelity wireframe for checkout" -m claude-haiku-4-5

交互式设计工作流

bash + shell 
pencil interactive -o design.pen

pencil > get_editor_state({ include_schema: true })
pencil > get_guidelines()
pencil > find_empty_space_on_canvas({ width: 1440, height: 900, padding: 100, direction: "right" })

# 创建页面结构
pencil > batch_design({ operations: 'hero=I(document,{type:"frame",name:"Hero",width:1440,height:900,fill:"#0A0A0A"})' })
pencil > batch_design({ operations: 'heading=I("hero",{type:"text",content:"Ship faster.",fontSize:72,fill:"#FFFFFF"})' })

# 验证效果
pencil > get_screenshot({ nodeId: "hero" })

# 导出并保存
pencil > export_nodes({ nodeIds: ["hero"], outputDir: "./exports" })
pencil > save()
pencil > exit()
