OpenCode CLI

版本：1.3.15 | 安装路径：/opt/homebrew/bin/opencode

OpenCode 是一个交互式 CLI 工具，提供 TUI（终端用户界面）和 headless 模式，用于与 AI 模型交互、管理会话、操作代码等。它支持多种 AI 提供商、MCP/ACP 协议、GitHub 集成和插件扩展。

快速开始

# 在当前目录启动 TUI 交互界面
opencode

# 在指定项目目录启动
opencode /path/to/project

# 直接运行一条命令（非交互模式）
opencode run "帮我写一个 hello world"

# 继续上一次的会话
opencode -c

# 查看版本
opencode -v

全局选项

选项说明默认值
-h, --help显示帮助信息-
-v, --version显示版本号-
--print-logs将日志输出到 stderrfalse
--log-level日志级别：DEBUG / INFO / WARN / ERROR-
--pure不加载外部插件运行false

网络选项

选项说明默认值
--port监听端口0（随机）
--hostname监听主机名127.0.0.1
--mdns启用 mDNS 服务发现false
--cors额外允许的 CORS 域名[]

核心命令

opencode — 启动 TUI

opencode [project]

启动 OpenCode 的终端交互界面（默认命令）

选项说明
-m, --model <provider/model>指定使用的模型
-c, --continue继续上一次会话
-s, --session <id>继续指定会话
--fork以 fork 模式继续会话
--prompt <text>指定系统提示词
--agent <name>指定使用的 Agent

opencode run — 非交互模式

# 简单执行
opencode run "解释这段代码的作用"

# 附加文件
opencode run "审查这个文件" -f src/main.ts

# JSON 输出（适合管道处理）
opencode run "列出所有 TODO" --format json

# 使用特定模型和推理强度
opencode run "重构这段代码" -m anthropic/claude-sonnet-4-20250514 --variant high

opencode serve — 启动服务器

# 启动服务，监听 8080 端口
opencode serve --port 8080

# 允许局域网访问
opencode serve --hostname 0.0.0.0 --port 4096

# 启用 mDNS 发现
opencode serve --mdns

opencode web — Web 界面

# 启动 Web 界面
opencode web --port 3000

# 局域网访问 Web 界面
opencode web --hostname 0.0.0.0 --port 3000 --cors http://myapp.local

会话管理

opencode session

子命令说明
session list列出所有会话
session delete <sessionID>删除指定会话

opencode export / import

# 将会话数据导出为 JSON
opencode export [sessionID]

# 从 JSON 文件或 URL 导入会话数据
opencode import <file>

opencode stats

显示 Token 使用和费用统计。

# 最近 7 天统计
opencode stats --days 7

# 按项目过滤
opencode stats --project "my-app"

# 显示模型统计
opencode stats --models

模型与提供商

opencode models

列出所有可用模型。

# 列出所有模型
opencode models

# 列出 anthropic 的模型
opencode models anthropic

# 详细模式
opencode models --verbose

# 刷新缓存
opencode models --refresh

opencode providers

子命令说明
providers list列出所有已配置的提供商
providers login [url]登录提供商
providers logout登出已配置的提供商

Agent 管理

opencode agent

子命令说明
agent list列出所有可用 Agent
agent create创建新的 Agent

创建 Agent 选项

选项说明
--path <dir>生成 Agent 文件的目录路径
--description <text>Agent 功能描述
--mode <all|primary|subagent>Agent 模式
--tools <list>启用的工具列表（逗号分隔）
-m, --model使用的模型

可用工具：bash, read, write, edit, list, glob, grep, webfetch, task, todowrite

MCP 服务器管理

opencode mcp

子命令说明
mcp add添加 MCP 服务器（交互式）
mcp list列出 MCP 服务器及其状态
mcp auth [name]对 OAuth 支持的 MCP 服务器进行认证
mcp logout [name]移除 MCP 服务器的 OAuth 凭证
mcp debug <name>调试 MCP 服务器的 OAuth 连接

插件管理

opencode plugin

# 安装插件
opencode plugin @opencode/plugin-example

# 全局安装
opencode plugin @opencode/plugin-example -g

# 强制替换
opencode plugin @opencode/plugin-example -f

GitHub 集成

opencode github

子命令说明
github install安装 GitHub Agent
github run运行 GitHub Agent

opencode pr

拉取并检出 GitHub PR 分支，然后启动 OpenCode。

# 检出 PR #42 并启动
opencode pr 42

配置文件

OpenCode 的配置文件位于 ~/.config/opencode/ 目录下。

路径类型默认路径说明
home~用户主目录
data~/.local/share/opencode数据目录（会话、快照等）
config~/.config/opencode配置文件目录
cache~/.cache/opencode缓存目录
log~/.local/share/opencode/log日志目录

使用 opencode debug config 命令查看完整的已解析配置。

本手册基于 OpenCode v1.3.15 生成，命令和选项可能随版本更新而变化。使用 opencode --help 查看最新信息。
